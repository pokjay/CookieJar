# Architecture

## Stack

| Layer | Technology |
|---|---|
| Frontend | Next.js 15, Tailwind CSS, Tremor, Recharts, ECharts (echarts-for-react), TanStack Table |
| Backend | FastAPI, Python 3.12, pandas, SQLAlchemy, Uvicorn |
| Database | PostgreSQL (`moneyman` schema), schema managed with dbmate |
| E2E tests | Playwright in Docker |
| Unit tests | pytest |

## Pages

| Route | Description |
|---|---|
| `/` | **Overview** — net worth metrics + YoY delta badges, pie chart by asset category, income vs. expense bars, net worth area chart per person, net worth by category line chart; year selector + person filter |
| `/cash-flow` | **Cash Flow** — household aggregate table (year expand/collapse), per-person cash flow, dual-axis area chart (income/expense + savings rate), Sankey diagram with click-to-expand subcategories |
| `/transactions` | **Transactions** — Dashboard tab: data health signals, YoY total bar, monthly YoY grouped bar, avg monthly spend by category, subscription detection, category trend grid, top businesses, day-of-week heatmap, uncategorized count; Browse tab: filterable/sortable TanStack table |
| `/travel` | **Travel** — same as Transactions but filtered to Travel category; subcategory breakdown; per-trip aggregation |
| `/category-mapping` | **Category Mapping** — lists uncategorized descriptions with count/total, form to assign category + subcategory |
| `/business-mapping` | **Business Mapping** — map broad transaction descriptions to specific business names |
| `/manual-transactions` | **Manual Transactions** — add a single transaction or bulk-import from CSV |
| `/settings` | **Settings** — configure family members (Parent 1, Parent 2, Kids), sign-flipped accounts, cash flow bank accounts, account-to-person mapping |
| `/investments` | **Investments** — investment accounts with latest valuations and manual data entry |

## Data Flow

```
Next.js (port 3001)
  └── /api/[...proxy]   → proxy route rewrites to FastAPI
        └── FastAPI (port 8083)
              └── backend/routers/   → domain-specific routers
                    └── src/db/      → shared DB access layer
                          ├── queries/    → read queries (pandas DataFrames)
                          ├── mutations/  → write operations
                          └── mock_data.py → deterministic fake data (mock mode)
```

## Backend Structure

```
backend/
  main.py              # FastAPI app, CORS, router registration
  cache.py             # 5-minute TTL cache (module-level dict, keyed by fn name)
  data.py              # Overview/investments business logic (net worth, cash flow aggregations)
  data_transactions.py # Transactions/travel business logic (YoY, subscriptions, heatmap, etc.)
  routers/
    overview.py        # GET /api/overview/*
    cash_flow.py       # GET /api/cash-flow/*
    transactions.py    # GET /api/transactions/*
    travel.py          # GET /api/travel/*
    categories.py      # GET/POST /api/categories/*
    business.py        # GET/POST /api/business/*
    manual_transactions.py  # GET/POST /api/manual-transactions/*
    investments.py     # GET /api/investments/*
    settings.py        # GET/POST /api/settings/*
```

## Shared Source Layer (`src/`)

```
src/
  constants.py         # Hebrew→English maps (account types, categories), color palettes
  settings.py          # Load/save app_settings.json (family members, account mappings)
  db/
    connection.py      # is_mock_mode(), run_query(), execute_mutation(), get_enum_values()
    mock_data.py       # Generates ~2200 transactions, 184 cash flow rows, 10 investment
                       # accounts, 150 tracking snapshots, 65 category mappings, etc.
    queries/           # Read-only SQL queries, return pandas DataFrames
    mutations/         # Write operations (INSERT / UPSERT)
  utils/
    calculations.py    # Data transformations: net worth, cash flow aggregation, Sankey prep, subscriptions
    formatting.py      # format_currency, format_percentage, format_delta
```

## Caching

All read paths in the backend are wrapped with `ttl_cached` (5-minute TTL, `backend/cache.py`). Each router declares module-level cached references, e.g.:

```python
_cash_flow = ttl_cached(get_all_cash_flow)
```

After any write (manual transaction insert), `clear_all()` evicts every cache entry so the next request gets fresh data.

## Mock Mode

`USE_MOCK_DATA=true` (the default) bypasses the database entirely. All `src/db/queries/` functions return DataFrames generated by `src/db/mock_data.py`. Mutations in mock mode write to a module-level in-process dict (not persisted across restarts). No database connection is required.

## Settings Persistence

App settings (family members, sign-flipped accounts, cash flow accounts, account-to-person mapping) are saved to `config/app_settings.json` and reloaded on each API call via `src/settings.py`.

## Cash Flow Derivation

When the `monthly_cash_flow` table has no data for a given month but `transactions_manual` does (for the configured bank accounts), the backend derives cash flow automatically from those transactions. For any month where `transactions_manual` data exists, it **replaces** the entire month from `monthly_cash_flow` — so all bank account transactions for that month must be present or the figures will be incomplete.

## Key Design Decisions

- **Hebrew translations** happen in Python (`src/constants.py`), not in SQL
- **Sign-flipped accounts**: accounts where debit amounts are stored as negatives; the backend flips them before analysis
- **Parameterised SQL** (`:param` syntax via SQLAlchemy) everywhere — no string interpolation in queries
- **`src/db/`** is shared between the FastAPI backend and `tests/`; it has no dependency on the frontend
- **Settings are read on every request** (no caching) so changes take effect immediately without a restart
